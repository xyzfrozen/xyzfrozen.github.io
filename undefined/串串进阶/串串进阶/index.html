Hash

对一种图形进行 $Hash$ 就是要体现这个图的结构，主要表现在 “相邻” 结构，从而隐式的表达图结构

树 Hash

对于一棵树，就是通过子树表达相邻

对于有根树，可以直接从根开始，无根树可以从重心开始，如果有两个，就都存下来比较

$Hash$ 函数：$H(x)=\sum_{v \in S} H(v) \times P^{\sum sz_v}$

图 Hash

除了树，还可以图 $Hash$

体现图结构就是通过相邻的点集进行传递，传递 $i$ 次就相当于距离为 $i$ 的点的结构贡献

$Hash_{x,j} = A \times Hash_{x,j-1}+B \times \sum_{(u,v) \in E} (Hash_{v,j-1}+j)$

这个 $j$ 取个 4，5 次或者直接 $O(n)$ 就差不多，每次迭代后排序 $Hash$ 值一一比对，$+j$ 就是类似字符串 $Hash$ 的 $+s_i$

每次错误概率是 $\frac 1p$ ，进行 $n$ 次相当于完整遍历图结构，$O(nm+n^2\log n)$

Problem

[P3167 CQOI2014] 通配符匹配

像通配的题, 我们通常设 $f_{i,j}$ 表示 $S$ 串到 $i$ 位, $T$ 串到 $j$ 位是否可行, 然后在$O(|S||T|)$ 的时间内解决

如何优化呢? 不难发现对于 $S$ 串来说, 其实很多段是可以连续转移的, 也就是那些有通配符的地方才有用

那么就可以设 $f_{i,j}$ 为 $S$ 中的第 $i$ 个通配符，$T$ 串到 $j$ 位是否可行,对于串尾的 $*/?$ 的转移单独考虑

这样的话, 每次转移会转移连续的一段, 我们就是要判断对于 $S$ 和 $T$ 对应的两段是否相同, 直接用哈希比较即可

$O(|S|+10\sum|T|)$

[题解 P3167 【CQOI2014]通配符匹配】 - Diu の blog - 洛谷博客 (luogu.com.cn)

[P4323 JSOI2016] 独特的树叶

首先叶子节点一定是入度为 $0$

我们直接删掉这个叶子节点，那么以他的父亲为根的树必然和 $A$ 树的某一个形态同构，可以用树 $Hash$ 来判断同构

求出 $A,B$ 两树以每个点为根的 $Hash$ 值，换根 $O(n)$，然后把 $A$ 的全部存在 set 里面，每个 $B$ 的每个叶子节点判断删掉它后父亲的值

这里就要设计一种方便 “撤销贡献” 的 $Hash$ 函数了，我们想到 xor

$Hash_x = \oplus (Hash_v \times rnd +sz_v)$

$rnd$ 选的是 99994919 主要是扩大值域，感觉这个函数不如 $Hash_x=\sum Hash_v \times prime(sz_v)$

Exkmp

给定两个字符串 $a,b$，你要求出两个数组：

  • $b$ 的 $z$ 函数数组 $z$，即 $b$ 与 $b$ 的每一个后缀的 LCP 长度。
  • $b$ 与 $a$ 的每一个后缀的 LCP 长度数组 $p$。

$O(n)$

我们考察一个增量算法，利用 $dp$ 来计算，类似 kmp

首先设 $z_i$ 表示 $lcp(b[i,m],b[1,m])$，显然有 $z_1=m$

从 $i=2$ 开始，我们首先维护一个 $[l,r]$ 满足 $l \lt i,\and,r=l+z_l-1$ 且 $r$ 最大

注意我们现在是要在 $i$ 后面完成增加，我们利用 $z_{i-l+1}$

则 $[i-l+1,p]=[i,k]=z_{i-l+1}$

并且我们可以反证得到 $s[p+1] \not =s[k+1]$ ，否则 $z_{i-l=1}$ 可以变大

但是这样有一个小问题，为什么 $k \leq r$ 呢？这是我们不知道的，但是我们知道 $z_i \geq r-i+1$

所以我们把 $z_i$ 赋值成 $\min (z_{i-l+1},r-i+1)$ 这样就可以了

但是还有一个问题，为什么 $i \leq r$ 呢，我们也不知道，同时上面赋值成 $\min$ 可能是会增大的

所以我们还要暴力让 $i$ 向后面匹配，由于匹配一定是会让 $r$ 增加，我们可以保证最多只会增加 $m$ 次

$O(m)$

设 $ext_i=lcp(a[i,n],b[1,m])$

这其实和求 $z_i$ 没有什么不同，我们把一些 $z$ 修正为 $ext$ 即可，修正那些呢？

我们要利用的是 $b$ 的前缀，所以需要给 $ext_i$ 赋值时用到 $z$

利用的是 $a[l,r]=[1,r-l+1]$ 再给 $i$ 后面增加一段，增加就是核心思想，推导时都可以从 $[l,r]$ 这个条件出发，再利用 $i-l+1$ 对应的 $z$

Manacher

和 Exkmp 非常像啊

首先这个算法有一个弊端就是只能求奇数回文串，所以我们想到经典 Trick，我们可以给每个字符之间插入一个 '#'，再给头尾各加一个，这样所有长度为 $L$ 的回文串就会变成长度为 $2L+1$ 的回文串，所以数组要开两倍！！

还是利用之前的信息，设 $i+p_i-1$ 最大的 $i$ 为 $mid$，$r=mid+p_{mid}-1$

  1. $i \leq r$ 设 $j$ 为 $i$ 关于 $mid$ 的对称点，则 $j=2\times mid-i$

    则 $p_i=p_j$ ，当然可能 $i+p_i-1 \gt r$ 了，我们就直接赋值 $p_i=\min(p_j,r-i+1)$ 即可

  2. $i \gt r$ 暴力增即可

同样 $O(n)$
